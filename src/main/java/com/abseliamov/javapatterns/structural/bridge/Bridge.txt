Bridge (Мост) - делает конкретные классы независимыми от классов реализации интерфейса.

Шаблон «Мост» — это предпочтение компоновки наследованию.
Подробности реализации передаются из одной иерархии другому объекту с отдельной иерархией.

Шаблон «Мост» означает отделение абстракции от реализации, чтобы их обе можно было изменять
независимо друг от друга.

Примеры применения паттерна проектирования Bridge в JDK:
В шаблоне DAO объект DAO, с точки зрения приложения, является мостом.
    Приложение использует интерфейс DAO, который не меняется при изменении СУБД.
Также архитектура Java AWT основана на этом паттерне.
Также можно указать как пример LinkedHashMap(LinkedHashSet<K>, List<V>) который возвращает
    немодифицируемый linked map, который не клонирует объекты, но использует их.
